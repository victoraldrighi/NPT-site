## Spacing Scale

Declared values — token names map to `tokens.css` `--space-*` variables:

| Token | CSS Var | Value | Usage |
|-------|---------|-------|-------|
| xs | `--space-1` | 4px | Icon gaps, inline micro-spacing |
| sm | `--space-2` | 8px | Form field label-to-input, inline tag gaps |
| md | `--space-4` | 16px | Default block margin, form field margin-bottom |
| lg | `--space-6` | 24px | Section inner padding, form group spacing |
| xl | `--space-8` | 32px | Layout column gaps, article padding |
| 2xl | `--space-12` | 48px | Major section top/bottom padding |
| 3xl | `--space-16` | 64px | Page-level vertical rhythm, hero and CTA section |

Exceptions:
- Touch targets for the floating WhatsApp button: 48px minimum (width and height) — accessibility minimum for a fixed action button
- Form `<input>` and `<select>` min-height: 44px (touch target floor for mobile)
- Form `<textarea>` min-height: 120px

Source: `tokens.css` primitives; touch target values per WCAG 2.5.5 (AAA advisory, 44px minimum)

## Layout Specifications

### /contato/ — Contact Page

**Mobile (375px base):** Single column. Contact info block above the form.
- Page `<h1>` "Solicitar Orçamento" at top
- Contact info block: phone (tel: link), email (mailto: link) — visible without scrolling on mobile
- Form below contact info
- CTA section (navy `.service-cta` pattern) at bottom with link to WhatsApp when activated

**Tablet (≥768px / 48rem):** Single column still acceptable. Form max-width 560px, centered.

**Desktop (≥1280px / 80rem):** 2-column grid.
- Left column (60%): form
- Right column (40%): contact info card + map placeholder (placeholder only — no map embed in v1)
- Max-width: `.container` (75rem)

### /portfolio/ — Portfolio Page

**All breakpoints:** Single column, case card centered, max-width 720px.
- Breadcrumb at top (reuse `.breadcrumb` from `service-page.css`)
- Page `<h1>` "Portfólio de Projetos"
- Intro paragraph (1-2 sentences)
- Single `.portfolio-case` card
- `.portfolio-cta` section (navy background, `.service-cta` class) at bottom

### Header — Phone number addition

**Desktop (≥80rem):** Phone number rendered as `<a href="tel:+5511900000000">` to the right of nav links, before or replacing the "Contato" nav item
**Mobile (inside hamburger menu):** Phone number as last `<li>` in `.nav-links` — same visibility behavior as other nav items (uses `visibility: hidden` on closed state per Phase 2 decision)

### Floating WhatsApp button

- Rendered in every page's HTML inside `<body>`, after `<main>`, before `</body>`
- Starts with `hidden` attribute
- When `hidden` removed: fixed bottom-right (56×56px circle), above-fold at all scroll depths
- z-index 900 (above page content, below hypothetical modals)

## Interaction Contract

### Form — Conditional "Outro" field

- When user selects "Outro" in the `<select>`, a free-text `<input type="text" name="tipo-outro">` appears
- Implementation: CSS `:has()` selector on the form — `.form-group--conditional { display: none }` by default; `.contact-form:has(select[name="tipo-projeto"] option[value="outro"]:checked) .form-group--conditional { display: block }`
- Fallback for browsers without `:has()` support: the text field is always visible (progressive enhancement — older browsers see both fields, modern browsers see conditional behavior)

### Form submission flow

1. User fills all required fields → clicks "Enviar Solicitação"
2. Browser native validation fires first (no JavaScript)
3. Netlify Forms intercepts POST on build-time registered form
4. Redirect to `/obrigado/` on success (existing page from Phase 2)
5. No JavaScript required for any step

### WhatsApp button activation path

- HTML ships with `hidden` attribute on `.whatsapp-btn` element
- To activate: remove `hidden` attribute, replace `55119XXXXXXXX` with real number in `href`
- No code changes beyond those two edits — no deploy script, no build step

### Focus management

- All form controls show focus ring: outline 2px `--color-brand-primary`, offset 2px
- Tab order: Name → Email → Phone → Project type → (conditional Other field if visible) → Message → Submit
- Form submit button is `type="submit"` (not `type="button"`) — activated by Enter key in any field

### Link accessibility

- Phone in header: `<a href="tel:+55119XXXXXXXX" aria-label="Ligar para NPT Engenharia: (11) 9 0000-0000">`
- Email in footer: `<a href="mailto:contato@nptengenharia.com.br">` — no extra aria-label needed (link text is descriptive)
- WhatsApp button when active: `aria-label="Falar pelo WhatsApp"`, icon `aria-hidden="true"`
