Note on font preload: CONTEXT.md originally required `<link rel="preload">` for Inter 400 in all pages. RESEARCH.md found that the font filename hash changes on every CSS rebuild, creating a maintenance risk (stale preload = double fetch = worse performance). Since `@fontsource/inter` already includes `font-display:swap` in all @font-face declarations (verified in compiled CSS), FOIT is already prevented. The preload is skipped in favor of verifying font-display:swap is present in the build output.

  <name>Task 3: Add cache and security headers to netlify.toml</name>
  <files>netlify.toml</files>
  <read_first>netlify.toml</read_first>
  <action>
The existing netlify.toml contains only build configuration:
```toml
[build]
  command = "npm run build"
  publish = "dist"

[build.environment]
  NODE_VERSION = "20"
```

Append the following header rules AFTER the existing build config. Do NOT remove the existing [build] and [build.environment] sections.

Append this exact content:

```toml

[[headers]]
  for = "/*"
  [headers.values]
    X-Frame-Options = "DENY"
    X-Content-Type-Options = "nosniff"

[[headers]]
  for = "/assets/*"
  [headers.values]
    Cache-Control = "public, max-age=31536000, immutable"
```

Why these specific rules:
- `/*` applies security headers (X-Frame-Options, X-Content-Type-Options) to ALL responses — including subdirectory pages like /sobre/, /contato/, /portfolio/, etc. This is critical: using `/*.html` or `/` only would miss subdirectory pages.
- `/assets/*` covers ALL Vite hashed assets (JS, CSS, fonts) — they are content-addressed so immutable caching is safe.
- Netlify header rules are additive: both rules apply to `/assets/` responses (security + cache), which is correct.
- HTML cache is not explicitly set because Netlify defaults to short cache for HTML, which is the desired behavior (revalidate on each visit).
