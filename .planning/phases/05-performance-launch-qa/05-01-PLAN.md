<task type="auto">
  <name>Task 3: Add cache and security headers to netlify.toml</name>
  <files>netlify.toml</files>
  <read_first>netlify.toml</read_first>
  <action>
The existing netlify.toml contains only build configuration:
```toml
[build]
  command = "npm run build"
  publish = "dist"

[build.environment]
  NODE_VERSION = "20"
```

Append the following header rules AFTER the existing build config. Do NOT remove the existing [build] and [build.environment] sections.

Append this exact content:

```toml

[[headers]]
  for = "/assets/*"
  [headers.values]
    Cache-Control = "public, max-age=31536000, immutable"

[[headers]]
  for = "/*.html"
  [headers.values]
    Cache-Control = "no-cache, no-store, must-revalidate"
    X-Frame-Options = "DENY"
    X-Content-Type-Options = "nosniff"

[[headers]]
  for = "/"
  [headers.values]
    Cache-Control = "no-cache, no-store, must-revalidate"
    X-Frame-Options = "DENY"
    X-Content-Type-Options = "nosniff"
```

Why these specific rules:
- `/assets/*` covers ALL Vite hashed assets (JS, CSS, fonts) — they are content-addressed so immutable caching is safe
- `/*.html` covers HTML files that Netlify serves at root level
- `/` explicitly covers the homepage route (Netlify serves index.html at `/`)
- Security headers X-Frame-Options: DENY prevents clickjacking; X-Content-Type-Options: nosniff prevents MIME sniffing
